# Assignment 3

## Тема: Архитектура GPU и оптимизация CUDA-программ

Данный проект выполнен в среде Google Colab с использованием графического процессора (GPU) и компилятора NVIDIA CUDA (nvcc). Все программы реализованы на языке C++ с применением технологии CUDA и предназначены для исследования архитектурных особенностей GPU и влияния различных методов оптимизации на производительность.

Размер обрабатываемых массивов во всех заданиях составляет 1 000 000 элементов.

Компиляция и запуск каждого задания осуществлялись в Google Colab с помощью команды:

```
!nvcc -arch=sm_75 task_X.cu -o task_X && ./task_X
```

где `X` — номер задания (1, 2, 3 или 4).

---

## Задание 1 (task_1.cu)

### Поэлементная обработка массива с использованием глобальной и разделяемой памяти

Реализованы две версии CUDA-программы умножения элементов массива на константу:

1. Версия с использованием только глобальной памяти GPU.
2. Версия с использованием разделяемой (shared) памяти.

Проведено сравнение времени выполнения для массива из 1 000 000 элементов.
Цель - продемонстрировать влияние использования shared memory на скорость доступа к данным и общую производительность.

Запуск:

```
!nvcc -arch=sm_75 task_1.cu -o task_1 && ./task_1
```

## Задание 2 (task_2.cu)

### Поэлементное сложение массивов и влияние размера блока потоков

Реализовано CUDA-ядро для поэлементного сложения двух массивов.
Выполнены замеры времени для нескольких размеров блока потоков:

* 128 потоков
* 256 потоков
* 512 потоков

Проанализировано, как размер блока влияет на загрузку мультипроцессоров, латентность доступа к памяти и общее время выполнения.

Запуск:

```
!nvcc -arch=sm_75 task_2.cu -o task_2 && ./task_2
```

## Задание 3 (task_3.cu)

### Коалесцированный и некоалесцированный доступ к глобальной памяти

Реализованы два варианта обработки массива:

1. С коалесцированным доступом (соседние потоки обращаются к соседним ячейкам памяти).
2. С некоалесцированным доступом (разреженный, несогласованный шаблон обращений).

Проведено сравнение времени выполнения для массива из 1 000 000 элементов и сделан вывод о влиянии шаблона доступа к памяти на пропускную способность GPU.

Запуск:

```
!nvcc -arch=sm_75 task_3.cu -o task_3 && ./task_3
```

## Задание 4 (task_4.cu)

### Подбор оптимальной конфигурации сетки и блоков потоков

В качестве базовой программы выбрано поэлементное сложение массивов из Задания 2.
Для неё исследованы две конфигурации:

* Неоптимальная: размер блока 32 потока
* Оптимизированная: размер блока 256 потоков

Выполнено сравнение времени выполнения и рассчитано ускорение, получаемое за счёт правильного выбора параметров сетки и блоков.
