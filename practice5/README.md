# Практическая работа: Параллельные структуры данных на CUDA

## Описание
Этот проект содержит две части:

### Часть 1: Параллельный стек на CUDA
- Реализован стек с использованием атомарных операций (`atomicAdd` и `atomicSub`) для безопасного параллельного доступа.
- Потоки GPU выполняют операции `push` и `pop` одновременно.
- Выводит:
  - Размер стека
  - Конфигурацию GPU (количество блоков и потоков)
  - Время выполнения ядра
  - Проверку корректности атомарных операций

### Часть 2: Параллельная очередь и стек на CUDA
- Реализована очередь и стек с атомарными операциями (`enqueue/dequeue` и `push/pop`).
- Замеряется время работы каждой структуры на GPU.
- Выводит:
  - Размер очереди/стека
  - Время работы очереди
  - Время работы стека
  - Сравнение производительности

## Запуск в Google Colab
1. Сначала убедитесь, что у вас включен GPU:
   - `Среда выполнения` → `Сменить среду выполнения` → `Аппаратный ускоритель GPU`

2. Загрузите файлы `part1.cu` и `part2.cu` в Colab (через значок файлов или drag & drop)

3. Компиляция и запуск:

### Часть 1: Параллельный стек
```bash
# Компилируем и запускаем
!nvcc -arch=sm_75 part1.cu -o part1 && ./part1
```
### Часть 2: Параллельная очередь и стек
```bash
# Компилируем и запускаем
!nvcc -arch=sm_75 part2.cu -o part2 && ./part2
```
